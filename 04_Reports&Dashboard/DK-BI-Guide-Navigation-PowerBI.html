🎯 Drill-through — Exemple : Géographie → Profil Commune

Le drill-through permet à l'utilisateur de cliquer-droit sur une commune dans un visuel de la page Géographie pour atterrir sur la page Profil Commune, pré-filtrée sur cette commune.

1
Crée la page détail "Profil Commune" et nomme-la correctement
Dans le panneau inférieur de Power BI Desktop, crée une nouvelle page et renomme-la exactement Profil Commune. C'est la page de destination du drill-through.

Donne un nom clair à tes pages détail — Power BI l'affiche tel quel dans le menu clic-droit. L'utilisateur verra "Explorer > Profil Commune".
2
Sur la page détail, ouvre le volet "Visualisations"
Clique sur la page Profil Commune (sans sélectionner de visuel). Dans le panneau Visualisations à droite, descends jusqu'à la section "Générer un rapport" puis cherche "Extraction".

🖱️ Boutons de Navigation — Exemple : Démographie → Analyse Centenaires

Pour les pages qui n'ont pas besoin de contexte (centenaires, mineurs, anomalies), on utilise des boutons avec une action de navigation directe.

1
Sur la page principale, insère un bouton vide
Dans la page Démographie, va dans le menu Insertion → Boutons → Vide. Un bouton rectangle apparaît sur la page — repositionne-le dans un coin ou une zone dédiée de ta mise en page.

↩ Bouton Retour — Revenir à la page principale

Chaque page détail doit permettre à l'utilisateur de revenir à la page principale. Power BI génère un bouton Retour automatiquement pour les drill-throughs, mais il faut en créer un manuellement pour les boutons de navigation.

1
Pour les drill-throughs : le bouton Retour est généré automatiquement
Dès que tu configures le puits d'extraction, Power BI insère automatiquement un bouton Retour en haut à gauche de la page détail. Ce bouton est pré-configuré avec l'historique de navigation — il ramène exactement là où l'utilisateur était.
Page "Profil Commune" — bouton Retour auto-généré
← Précédent
Profil Commune — Lyon (69123)
Ce bouton ← est généré automatiquement par Power BI lors du drill-through
Tu peux repositionner et reformater ce bouton (couleur, taille, texte). Il gardera son comportement de retour automatique.
2
Pour les boutons de navigation : crée un bouton Retour manuel
Sur chaque page détail accessible par bouton (Centenaires, Mineurs, Anomalies…), insère un bouton Insertion → Boutons → Précédent. C'est un bouton intégré Power BI qui utilise l'historique de navigation — pas besoin de configurer la destination.

3
Standardise le placement du bouton Retour sur toutes les pages détail
Place le bouton Retour au même endroit sur toutes tes pages détail — idéalement en haut à gauche, avec le fil d'Ariane. Ça crée une navigation cohérente et professionnelle.

Erreur fréquente n°1 — Le drill-through ne s'affiche pas dans le menu clic-droit.

Cause : le visuel sur la page source ne contient pas le champ mis dans le puits d'extraction. Vérifie que le même champ (code_insee_commune) est présent dans l'axe ou les détails du visuel.
Erreur fréquente n°2 — La page détail s'ouvre sans filtre.

Cause : le toggle "Conserver tous les filtres" est désactivé, ou le champ dans le puits ne correspond pas au champ dans les mesures DAX (ex: nom_commune vs code_insee_commune).
Bonne pratique pro — Masque les pages détail de la navigation principale.

Clic-droit sur l'onglet de la page détail → "Masquer la page". La page reste accessible via drill-through ou bouton, mais n'apparaît pas dans la barre de navigation normale du rapport publié.
Conseil portfolio — Ajoute un fil d'Ariane sur chaque page détail (ex: "Géographie › Profil Commune › Lyon"). Ça montre une vraie maîtrise de l'UX Power BI, rarement vue dans les portfolios juniors.
